# Clash / Mihomo 独立排障指南（按症状排查）

最近更新：2026-03-11

这篇文章只做一件事：**别再盲目换节点了，先按症状排查。**

很多问题看起来像“机场不行”，其实根本原因是下面四类之一：

- 模式不对；
- 系统代理或 `TUN` 没接管到目标应用；
- DNS 有问题；
- 订阅、规则集或节点地区不适合你的目标服务。

如果你已经导入成功，但体验不稳定，建议按本文顺序查，不要上来就连续切十几个节点。

---

## 先记住这 4 个高频结论

### 1. 日常默认先用 `Rule`

`Global` 适合临时排障，不适合长期默认。日常大多数场景先用 `Rule`。

### 2. 浏览器能用，不代表所有 App 都能用

系统代理通常先覆盖浏览器和部分桌面软件，很多 App、游戏和系统服务并不会自动跟随。

### 3. “能连上却打不开网页” 优先怀疑 DNS

尤其是遇到：

- 节点测速正常；
- 软件显示连接成功；
- 页面一直转圈或直接超时。

### 4. AI / 流媒体问题，经常不是客户端问题

`YouTube` 能开、`ChatGPT` 不能开，这类问题很多时候是节点地区、共享 IP 信誉或账号风控，而不是客户端坏了。

---

## 症状 1：客户端显示已连接，但所有网站都打不开

按顺序检查：

1. 当前模式是不是误开成了 `Direct`；
2. 系统代理是否真的开启；
3. 系统时间和时区是否正确；
4. DNS 是否异常；
5. 该节点是否真的可用。

你可以这样快速验证：

- 切到一个明确可用的节点；
- 用 `Rule` 模式；
- 打开 GitHub 或 YouTube；
- 如果还是完全打不开，再考虑换线路或换地区。

常见原因：

- DNS 污染或解析失败；
- 规则集过期；
- 系统时间不准导致 TLS 校验失败；
- 你以为代理开了，其实系统代理没有生效。

---

## 症状 2：浏览器能打开，Telegram / Discord / ChatGPT 客户端不行

这类问题优先怀疑**流量没有被接管到目标 App**。

排查顺序：

1. 先确认浏览器端确实正常；
2. 关闭浏览器代理插件，避免和客户端重复代理；
3. 启用 `TUN`；
4. 检查系统防火墙或安全软件是否拦截；
5. 重新启动目标应用，必要时清理 DNS 缓存。

典型判断：

- 浏览器正常，App 不正常：优先查 `TUN`；
- Windows 端不正常，iPhone 正常：优先查系统代理、权限、驱动。

---

## 症状 3：YouTube 能开，ChatGPT / Claude 不行

这通常不是“代理没通”，而是下面几类问题：

- 节点地区不在目标服务支持范围；
- 共享 IP 被滥用，信誉太差；
- 账号本身触发了风控；
- 你频繁切换地区，导致验证变多。

正确处理方式：

1. 先确认目标服务支持哪些地区；
2. 优先换“同地区不同节点”，不要先乱跳国家；
3. 少做高频切换；
4. 必要时退出重新登录，排除账号缓存问题。

经验上：

- `ChatGPT` / `Claude` 更看地区和 IP 信誉；
- `YouTube` 只要基础代理连通，通常更容易成功。

---

## 症状 4：节点延迟很低，但速度还是慢

低延迟只说明“握手不慢”，不代表真实吞吐高。

常见原因：

- 晚高峰拥堵；
- 线路超售；
- 回程质量差；
- 你选的是“适合网页、不适合下载”的线路；
- 测速节点和实际访问服务的地区不一致。

正确做法：

1. 不要只换节点名，先换线路类型；
2. 同地区里优先比较中转、专线、不同入口；
3. 分白天和晚高峰分别测；
4. 目标是视频，就直接测视频；目标是 AI，就直接测 AI。

---

## 症状 5：iPhone 可用，但电脑不行

这是很常见的一类问题，因为 iPhone 上多数客户端是通过 VPN 模式接管，而桌面端常常依赖系统代理或 `TUN`。

重点排查：

- Windows / macOS 是否开启系统代理；
- 桌面端是否授予了 `TUN` 所需权限；
- 是否被防火墙、杀毒软件拦截；
- 是否下错了客户端架构版本；
- 是否有其他代理软件占用了相同端口。

如果手机端一直稳定，而电脑端一直异常，通常先不要急着怪服务商，先把本机配置查干净。

---

## 症状 6：更新订阅失败

先看下面几个点：

1. 订阅是否过期；
2. 机场官网是否能打开；
3. 客户端版本是否过旧；
4. 当前节点或当前网络是否会影响访问面板域名；
5. 订阅地址是否被你误删、复制不完整或混入空格。

最直接的验证方式：

- 先在浏览器中打开官网登录页；
- 再看客户端能否更新订阅；
- 如果浏览器也打不开，优先检查面板域名是否变更。

---

## 症状 7：导入订阅后没有节点，或者策略组不完整

这种问题常见于：

- 订阅格式不完整；
- 客户端版本过旧，不支持新字段；
- 面板导出方式选错了；
- 你导入的是错误类型的订阅。

建议操作：

1. 换一个主流客户端再导入一次；
2. 更新到较新的 `mihomo` 系客户端；
3. 检查后台是否有“通用订阅”、“Clash / Mihomo 订阅”之分；
4. 不要用早期停更客户端硬吃新配置。

---

## 症状 8：游戏、下载器或某些桌面软件完全不走代理

重点看三件事：

- 是否启用了 `TUN`；
- 是否允许 UDP；
- 是否存在进程级代理需求。

如果目标是游戏：

- 先看节点是否适合 UDP；
- 再看晚高峰抖动和掉包；
- 不要只看延迟截图。

如果目标是下载器或开发工具：

- 看它是否跟随系统代理；
- 不跟随时再考虑 `TUN` 或应用自身代理设置。

---

## 5 分钟最小闭环

当你完全不知道从哪里开始时，用这 5 步重置思路：

1. 更新订阅；
2. 切到一个已知可用节点；
3. 模式设为 `Rule`；
4. 开系统代理，必要时开 `TUN`；
5. 先测 GitHub / YouTube，再测你的目标服务。

如果这 5 步都做完，基础站点仍然完全打不开，再考虑：

- 当前服务商线路是否普遍故障；
- 是否需要更换地区或更换服务。

---

## 什么时候该换服务商，而不是继续折腾客户端？

满足下面几条中的多条时，再考虑换服务商更合适：

- 多个客户端表现一致异常；
- 多个网络环境下都异常；
- 多个地区节点都不稳定；
- 晚高峰长期不可用；
- 官网、公告、客服同时失联。

如果只是单一服务打不开、单一设备不正常，通常优先排查本地配置，而不是立刻换机场。
